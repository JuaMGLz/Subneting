División de Redes (Subnetting) para Principiantes

Aprende a dividir una red grande en redes más pequeñas de forma manual y sencilla. ¡Sin programas, solo con tu cerebro!

¿Qué es una dirección IP?

Una dirección IP es como la dirección de tu casa, pero para un dispositivo en una red. Se compone de 4 números del 0 al 255, separados por puntos. Ejemplo: 192.168.10.15
La dirección IP se divide en dos partes:
  • Parte de Red: Es como el nombre de la calle. Es la misma para todos los dispositivos en la misma red.
  • Parte de Host: Es como el número de la casa. Es única para cada dispositivo en esa red.

Conceptos Clave que Necesitas Saber

Máscara de Subred

Es la herramienta que te dice qué parte de la IP es la "Parte de Red" y qué parte es la "Parte de Host". Se compone de una serie de unos seguidos de ceros.

Subred (Subnet)

Es una red más pequeña que se crea a partir de una red principal más grande. Esto ayuda a organizar y gestionar la red de forma más eficiente.

Host

Cualquier dispositivo que se conecta a una red, como una computadora, un teléfono o una impresora.

Dirección de Red

Es la primera dirección en una subred. Sirve para identificar la red en su totalidad y no se puede asignar a ningún dispositivo (host).

Dirección de Broadcast

Es la última dirección en una subred. Se usa para enviar un mensaje a todos los dispositivos que están en esa subred al mismo tiempo. No se puede asignar a un host.

Paso a Paso: El Método Manual

Vamos a usar un ejemplo: queremos dividir la red 192.168.1.0 con una máscara de subred 255.255.255.0 en 4 subredes más pequeñas.
1

Conviértelo a Binario

Para hacer los cálculos, necesitamos convertir la máscara de subred (255.255.255.0) a binario (ceros y unos). Cada 255 es un octeto (8 bits) lleno de unos, y el 0 son 8 ceros.

255.255.255.0
11111111.11111111.11111111.00000000

Contamos los unos, lo que nos da el CIDR. En este caso, hay 24 unos, así que es un /24.

2

Calcula la cantidad de Subredes

La fórmula es 2n, donde 'n' es el número de bits que vamos a "tomar prestados" de la parte de Host. Necesitamos 4 subredes, así que buscamos la potencia de 2 que sea igual o mayor a 4.

2^1 = 2 (No alcanza)
2^2 = 4 (¡Exacto!)

Necesitamos 2 bits prestados.

3

Calcula la cantidad de Hosts

Ahora, la fórmula es 2h - 2, donde 'h' es el número de bits restantes en la parte de Host. Nuestra máscara original tenía 8 bits de Host, y prestamos 2, así que nos quedan 6.

8 - 2 = 6 bits restantes
2^6 - 2 = 64 - 2 = 62 hosts por subred

El '- 2' es porque una dirección se usa para la red y otra para el broadcast (mensaje a todos los hosts).

4

Calcula el Salto de Red

Para encontrar la dirección de la primera subred, sumamos un número mágico a nuestra dirección original. Este número se llama "salto" y se calcula con la fórmula 256 - Máscara Nueva.

Primero, calculamos la Máscara Nueva. La parte de Red tiene 24 unos, y le sumamos los 2 que prestamos. Son 26 unos. Para saber qué número es, sumamos los valores de los bits que están en "1".

11111111.11111111.11111111.11000000
Máscara anterior: 255.255.255.0
Valores: 128 + 64 = 192
Máscara nueva: 255.255.255.192
Salto = 256 - 192 = 64

Este es el "salto". Cada subred comenzará a 64 números de la anterior.

5

Encuentra las direcciones de cada Subred

Usando el salto (64), podemos encontrar las direcciones de red, broadcast y el rango de hosts utilizables para cada subred.

Subred 1:
Dirección de Red: 192.168.1.0
Primer Host: 192.168.1.1
Último Host: 192.168.1.62
Dirección de Broadcast: 192.168.1.63
Subred 2: (Sumamos el salto: 0 + 64)
Dirección de Red: 192.168.1.64
Primer Host: 192.168.1.65
Último Host: 192.168.1.126
Dirección de Broadcast: 192.168.1.127
Subred 3: (Sumamos el salto: 64 + 64)
Dirección de Red: 192.168.1.128
Primer Host: 192.168.1.129
Último Host: 192.168.1.190
Dirección de Broadcast: 192.168.1.191
Subred 4: (Sumamos el salto: 128 + 64)
Dirección de Red: 192.168.1.192
Primer Host: 192.168.1.193
Último Host: 192.168.1.254
Dirección de Broadcast: 192.168.1.255

Resumen de los Pasos Sencillos

Paso 1: Binario

Convierte la máscara de subred a binario para encontrar los unos.

Paso 2: Subredes

Usa la fórmula 2n para saber cuántos bits "prestar" (n) para obtener el número de subredes que necesitas.

Paso 3: Hosts

Usa la fórmula 2h - 2 para saber cuántos hosts (h) caben en cada subred.

Paso 4: Salto

Calcula el salto con la fórmula 256 - Máscara Nueva para encontrar los puntos de inicio de cada subred.

Paso 5: Direcciones

Usa el salto para encontrar las direcciones de red, broadcast y el rango de hosts de cada subred.
